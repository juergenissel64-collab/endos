# Contributing to ENDOS

ENDOS is currently in an early foundation stage. Contributions should support clarity, traceability and architectural quality.

## Contribution Principles

- Preserve the integrity of the architecture baseline.
- Document assumptions, uncertainties and open questions.
- Keep changes modular and traceable.
- Prefer small, reviewable changes over large unstructured edits.
- Distinguish clearly between facts, hypotheses, design decisions and future ideas.

## Suggested Workflow

1. Create or identify the relevant sprint.
2. Describe the intended change.
3. Update documentation where needed.
4. Test the GitHub Pages output.
5. Record the change in `CHANGELOG.md` or release notes.

## Current Focus

The current priority is not feature quantity, but platform quality, structure and evidence readiness.
